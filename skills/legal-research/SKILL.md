# 律师法律研究 Skill

## 概述

专为律师设计的法律问题深度研究工具。通过并行启动多个独立研究 Agent，对复杂法律问题进行多维度、系统化的深度检索和分析，生成结构化的法律研究报告和应对策略文档。

## 核心特点

- **并行深度研究**：同时启动多个独立 Agent，每个负责一个研究课题
- **去重检索机制**：确保各 Agent 检索方向不重复，覆盖不同角度
- **法律专业性**：聚焦法律条文、司法解释、案例分析
- **实务导向**：提供可执行的应对策略和行动清单
- **文档输出**：自动生成 Word 和 PDF 格式的专业法律文档

## 适用场景

- 复杂法律纠纷的多维度分析
- 法律制度的系统性研究
- 诉讼策略的制定和论证
- 法律意见书的准备
- 案例研究和法律检索

## 使用方法

### 方式一：直接描述法律问题

```
/legal-research

[描述法律问题和案件背景]
```

### 方式二：提供文件

```
/legal-research @案件材料.md
```

### 方式三：指定研究课题

```
/legal-research

项目：[案件名称]
研究课题：
1. [法律基础研究]
2. [程序问题分析]
3. [实务策略制定]
```

## 处理流程

### 阶段1：分析准备
1. 读取法律问题和案件背景
2. 自动提取或确定研究课题
3. 课题拆分原则：
   - 方向明确：每个课题对应独特的检索方向
   - 避免重叠：确保课题之间检索关键词不重复
   - 聚焦问题：每个课题解决一个具体法律问题

### 阶段2：输出目录设置
按优先级检测项目结构：
1. 优先：`output/[项目名]/`
2. 其次：`./[项目名]/`
3. 兜底：`./legal-research/`

创建目录：`[输出目录]/03 - 🔍 深度研究/`

### 阶段3：并行深度研究
为每个研究课题启动独立的 `general-purpose` Agent：

**上下文传递**（主 Agent → 研究 Agent）：
- 案件关键信息（背景、争议焦点、核心问题）
- 完整的课题清单及各课题的检索范围
- 已分配的关键词方向（避免重复）
- 具体法律需求

**深度检索要求**：
- 4-6轮深度检索
- 使用 WebSearch（搜索发现）和 WebFetch（获取完整内容）
- 关键词差异化，确保每个 Agent 覆盖独特角度
- 重点检索：法律条文、司法解释、指导案例、学术观点

**文档生成**：
- 聚焦解决单一核心法律问题
- 简洁明了，提供明确法律意见
- 包含法律依据和案例引用
- 提供可执行的操作建议

### 阶段4：整合输出
1. 生成研究总览文档（000.研究总览.md）
2. 整合所有研究 Agent 的核心发现
3. 提供综合法律意见和应对策略
4. 创建立即行动清单

### 阶段5：文档转换
自动生成专业格式文档：
1. 合并所有研究报告为完整版 Markdown
2. 转换为 Word 文档（legal 预设格式）
3. 转换为 PDF 文档
4. 保存到桌面或指定位置

## 链接规范

**所有来源链接必须内嵌到正文中相应位置**

```markdown
✅ 正确：
根据[《民法典》第447条](链接)规定...

❌ 错误：
根据民法典第447条规定...
（文末单独列出引用来源）
```

## 文档命名规范

- `000.` - 研究总览
- `YYMMDD [课题名称].md` - 详细研究报告
- 使用简洁中文标题，15字以内
- 避免特殊字符

## 质量标准

### 研究质量
- **聚焦单一问题**：每个 Agent 仅解决一个核心法律问题
- **检索深度**：4-6轮检索
- **法律专业性**：引用法律条文、司法解释、指导案例
- **实务可操作性**：提供具体的操作步骤和时限要求

### 文档质量
- **结构清晰**：章节标题层级分明
- **叙述连贯**：段落式叙述，避免过度罗列
- **链接准确**：所有链接内嵌到正文相应位置
- **格式统一**：遵循统一的格式规范

## 依赖工具

### 内置工具（无需配置）
- **WebSearch**：搜索发现
- **WebFetch**：获取完整内容
- **Task**：启动独立研究 Agent

### 外部工具（需安装）
- **md2word skill**：Markdown 转 Word
  - 位置：`~/.claude/skills/md2word`
  - 依赖：`pip install python-docx Pillow beautifulsoup4 PyYAML`

- **docx2pdf**：Word 转 PDF
  - 安装：`pip install docx2pdf`

## 使用示例

### 示例1：租赁合同纠纷

**输入**：
```
租赁合同出租人有没留置权？出租房屋做商业酒店，现因欠交租金解除合同，
判决支付欠缴租金500万元，且查封了设备和装修。但属于二次查封，首封案件
属于工程款案件，不同意解封。出租人现在对于嵌入式厨房设备和空调外机，
有么有办法留置？写一篇应对策略文章，维护出租人权益
```

**自动生成研究课题**：
1. 租赁合同出租人留置权的法律基础
2. 二次查封与债权优先顺序分析

4. 出租人权益保护实务策略

**输出**：
- 4份详细研究报告（共67KB）
- 1份研究总览（含核心发现、综合建议、行动清单）
- Word 文档（legal 格式）
- PDF 文档

## 注意事项

### 严禁行为
- ❌ 不创建二级子目录
- ❌ 不生成独立的执行总结文件
- ❌ 不使用过度罗列的要点格式
- ❌ 不在文末单独列出引用来源

### 推荐做法
- ✅ 使用叙述式段落表达
- ✅ 链接内嵌到相应正文位置
- ✅ 保持简洁的研究总览
- ✅ 提供具体的行动建议和时限
- ✅ 标注清晰的文档编号

## 技术实现

### 核心脚本
本 skill 主要通过 Claude Code 内置工具实现，无需额外脚本。

### 工作流程
1. 分析法律问题 → 提取研究课题
2. 并行启动多个 general-purpose Agent
3. 每个 Agent 进行 4-6 轮深度检索
4. 生成结构化研究报告
5. 整合生成研究总览
6. 转换为 Word 和 PDF 文档
